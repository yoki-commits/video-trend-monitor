# 视频数据趋势监测应用 — 设计文档

## 1. 概述

构建一个基于 Flask + Chart.js + XLSX 的 Web 应用，用于长期监测抖音/抖音精选视频账号的数据变化趋势。用户通过上传两个固定的 .xlsx 格式数据文件（流量数据 + 粉丝数据），系统自动解析并持久化到数据库 XLSX 文件中，提供可视化的趋势分析和视频横向对比。

## 2. 技术栈

| 层级 | 技术 |
|------|------|
| 后端框架 | Python Flask |
| 数据处理 | pandas, openpyxl |
| 数据库 | `.xlsx` 文件作为数据库表 |
| 前端 | 原生 HTML + Chart.js CDN + CSS |
| 运行环境 | Python 3 + 浏览器 |

## 3. 数据结构

### 3.1 上传文件格式

用户每次上传两个文件，字段如下：

**流量数据.xlsx**
- Sheet `指标数据`: OGC_FID, 播放量, 点赞量, 评论量, 分享量, 收藏量, 弹幕量, 完播率, 2s跳出率
- Sheet `播放量-新增-每小时趋势数据`: OGC_FID, 日期, 播放量, 抖音, 抖音精选

**粉丝数据.xlsx**
- Sheet `指标数据`: OGC_FID, 涨粉量, 脱粉量, 粉丝播放占比
- Sheet `涨粉量-新增-每小时趋势数据`: OGC_FID, 日期, 涨粉量, 抖音, 抖音精选

### 3.2 数据库 XLSX 结构（`data_trend_db.xlsx`）

#### Sheet 1：`视频汇总`

| 字段 | 类型 | 来源 |
|------|------|------|
| video_id | 整数（自增） | 系统分配 |
| 发布日期 | 日期 | 用户选择 |
| 视频主题 | 文本 | 用户输入 |
| 上传时间 | 时间戳 | 系统自动 |
| OGC_FID | 整数 | 上传文件 |
| 播放量 | 整数 | 流量指标数据 |
| 点赞量 | 整数 | 同上 |
| 评论量 | 整数 | 同上 |
| 分享量 | 整数 | 同上 |
| 收藏量 | 整数 | 同上 |
| 弹幕量 | 整数 | 同上 |
| 完播率 | 文本 | 同上 |
| 2s跳出率 | 文本 | 同上 |
| 涨粉量 | 整数 | 粉丝指标数据 |
| 脱粉量 | 整数 | 同上 |
| 粉丝播放占比 | 文本 | 同上 |

#### Sheet 2：`播放量趋势`

字段完全沿用上传文件：OGC_FID, 日期, 播放量, 抖音, 抖音精选

#### Sheet 3：`涨粉量趋势`

字段完全沿用上传文件：OGC_FID, 日期, 涨粉量, 抖音, 抖音精选

## 4. API 设计

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/` | 首页 — 概览看板 |
| GET | `/upload` | 上传页面 |
| POST | `/api/upload` | 接收两个 xlsx 文件 + 表单，解析并存入数据库 |
| GET | `/api/videos` | 获取所有视频汇总列表（JSON） |
| GET | `/api/video/<id>` | 获取单视频详情 + 趋势数据（JSON） |
| GET | `/api/trends` | 获取累计趋势数据（JSON） |
| POST | `/api/video/<id>/delete` | 删除某期视频及其趋势数据 |
| GET | `/video/<id>` | 视频详情页面 |

## 5. 页面设计

### 5.1 上传页（`/upload`）

表单包含：
- 发布日期选择器
- 视频主题文本输入
- 两个文件上传控件（流量数据.xlsx、粉丝数据.xlsx）
- 提交按钮，上传成功跳转至看板

### 5.2 概览看板（`/`）

展示内容：
- 顶部 4 个统计卡片（总视频数 / 平均播放量 / 总涨粉 / 平均完播率）
- 各期视频播放量对比柱状图（Chart.js）
- 累计趋势折线图（播放量/涨粉随时间变化，Chart.js）
- 视频列表表格（主题、播放量、点赞、发布日期、操作按钮）

### 5.3 视频详情页（`/video/<id>`）

展示内容：
- 视频元信息（主题、发布日期）
- 全部指标卡片（2 行，覆盖 10 个指标）
- 播放量小时级趋势图（双线：抖音 vs 抖音精选）
- 涨粉量小时级趋势图
- 删除按钮（确认后删除）

## 6. 数据流

```
用户上传两个 .xlsx
    │
    ▼
Flask 接收文件 + 表单数据
    │
    ▼
pandas 读取流量.xlsx 的"指标数据" Sheet
pandas 读取粉丝.xlsx 的"指标数据" Sheet
    │
    ▼
合并数据 → 追加写入 data_trend_db.xlsx → "视频汇总" Sheet
    │
    ▼
pandas 读取趋势 Sheet → 写入 data_trend_db.xlsx → "播放量趋势" / "涨粉量趋势" Sheet
    │
    ▼
原始文件保存至 uploads/ 目录备份
    │
    ▼
返回成功 → 前端跳转至看板
```

读取流程：

```
前端请求 → Flask API
    │
    ▼
pandas.read_excel('data_trend_db.xlsx', sheet_name='视频汇总')
    │
    ▼
JSON 返回 → Chart.js 渲染图表
```

## 8. 边界情况处理

- **重复上传**：若 OGC_FID 已存在，提示用户该视频已存在，需确认是否替换
- **文件格式错误**：上传非预期格式的文件时，给出明确错误提示
- **删除确认**：删除操作需二次确认，删除后刷新数据
- **空数据状态**：首次使用无数据时，显示引导提示"请上传第一期视频数据"
